Security posture

Guardrails

  • Teller token encryption is application-layer before Oracle writes.
  • Amex credentials are entered only inside Teller Connect.
  • Teller enrollment signatures are verified with ED25519 key material.
  • Budget tables are isolated with the `BUDGET_` prefix.
  • Budget users are separated by `USER_ID` on Teller connections, accounts, transactions, and sync events.
  • Email verification and password reset tokens are stored as one-way hashes and expire automatically.
  • Registration and password reset responses avoid revealing whether an email exists.
  • Deploy behind HTTPS before exposing beyond localhost.
